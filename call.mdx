### Body

<ParamField body="to" type="string" required>
  This is the number to call.
</ParamField>

<ParamField body="context" type="string" default="">
  This controls the AI's objective and all the information around the objective.
</ParamField>

<ParamField body="from" type="string" default="Uses our credentials if empty.">
  This is the number to call from. Ensure this number is owned by your Twilio
  account.
</ParamField>

<ParamField
  body="twilioAccountSid"
  type="string"
  default="Uses our credentials if empty."
>
  This is your Twilio's AccountSid.
</ParamField>

<ParamField
  body="twilioAuthToken"
  type="string"
  default="Uses our credentials if empty."
>
  This is your Twilio's AuthToken.
</ParamField>

<ParamField body="forwardingPhone" type="string" default="null">
  This is the number AI will transfer to if it runs into any issues.
</ParamField>

<ParamField body="callbackUrl" type="string" default="null">
  This is the URL of your backend for function calling and call status updates.
</ParamField>

<ParamField body="voice" type="string" default="drew">
  This is the voice that will be used. Options are `drew`, `joseph`, `sarah`,
  `mark`, `maya`, and `patrick`.
</ParamField>

<ParamField body="model" type="string" default="gpt-4">
  This is the OpenAI model to use. Options are gpt-3.5-turbo, gpt-3.5-turbo-16k,
  gpt-4, and gpt-4-32k. GPT-4 is strongly recommended.
</ParamField>

<ParamField body="functions" type="array" default="null">
  These are the functions that the AI can invoke during the call.

  <Expandable title="properties">

    <ResponseField name="name" type="string">
      The name of the function to be called. Must be a-z, A-Z, 0-9, or contain
      underscores and dashes, with a maximum length of 64.
    </ResponseField>

    <ResponseField name="description" type="boolean">
      A description of what the function does, used by the AI to choose when and how
      to call the function.
    </ResponseField>

    <ResponseField name="parameters" type="object">
      The parameters the functions accepts, described as a JSON Schema object. See the [OpenAI guide](https://platform.openai.com/docs/guides/gpt/function-calling) for examples, and the [JSON Schema reference](https://json-schema.org/understanding-json-schema) for documentation about the format.

      To describe a function that accepts no parameters, provide the value `{"type": "object", "properties": {}}`.
    </ResponseField>

  </Expandable>
</ParamField>

<ParamField body="startTalking" type="boolean" default="false">
  This controls whether AI speaks first.
</ParamField>
